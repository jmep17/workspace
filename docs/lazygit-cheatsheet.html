Cherry-picking

  1. In any commits view (including another branch's, via Branches → enter), press C on each commit to copy it (range-select with v works too).
  2. Checkout the target branch.
  3. In its commits panel press V to paste. esc clears the copied set.

Custom patches — "how to patch"

Pull specific files or lines out of an existing commit (split a commit, move code between commits, extract to index):

  1. Commits panel → enter on the commit to see its files.
  2. Build the patch:
    • space on a file → toggle the whole file into the patch (green = whole file, yellow = partial), or
    • enter on a file → select lines (v for ranges, a for hunks) and space to toggle them into the patch.
  3. Press ctrl+p to open the custom patch options menu:
    • Remove patch from commit — deletes those changes from the commit (they vanish entirely)
    • Move patch out into index — pulls the changes out of the commit and leaves them staged
    • Move patch into new commit — splits the commit, placing the patch in a new commit after it
    • Move patch to selected commit — select another commit first, then move the changes into it
    • Apply patch / apply in reverse — apply to the working tree
    • Copy patch to clipboard

If a step causes conflicts, resolve them (see below) and continue with m. ctrl+p also shows "reset patch" to start over.

Undo only some lines in a file from a previous commit

You committed a file a few commits ago, and some of those lines were a mistake — but the rest of the commit is fine. This uses the custom patch builder to surgically pull just those lines back out.

  1. Go to the commits panel (4) and select the commit that introduced the bad lines. (Tip: ctrl+s → filter by path, or lazygit -f <file>, helps find which commit touched the file.)
  2. Press enter to see the commit's files, select the file, and press enter again to view its diff.
  3. Select only the lines you want to undo: move with j/k, press v to start a range selection, then press space to add the selected lines to the custom patch. (a grabs a whole hunk; space again toggles lines back out; d removes lines from the patch.)
  4. Press esc to step back, then press ctrl+p to open the custom patch options. Pick one:
    • Remove patch from commit — the lines are undone as if never committed. History is rewritten; the rest of the commit stays intact.
    • Move patch out into index — the lines leave the commit but land as staged changes, so you can inspect, tweak, or drop them yourself.
    • Apply patch in reverse — reverts those lines in your working tree only, without rewriting the commit. Use this if the branch is already shared/pushed and you'd rather commit a small revert.
  5. If later commits also touched those lines you'll get conflicts — resolve them in the conflicts view, then continue with m.
⚠️ The first two options rewrite history (an interactive rebase under the hood), so you'll need a force-push (P, lazygit offers --force-with-lease) if the commit was already pushed. Avoid on shared branches — use apply in reverse or t (revert) instead. And if it goes sideways: z undoes it.

Handy workflows

  • Amend an old commit: stage the changes (files panel) → commits panel → select the commit → A. Or ctrl+f in the files panel to auto-find the target and create a fixup.
  • Split the last commit: commits panel → g → soft reset to the commit before → restage and commit in pieces.
  • Undo a bad rebase: z (repeatedly) — it walks the reflog.
  • See what git commands lazygit runs: @ → toggle command log; great for learning the underlying git.
  • Diff two branches: W on the first ref → move to the second → W → view diff; esc exits diff mode.
  • Bisect: commits panel → b on a bad commit → mark good/bad as it steps.
